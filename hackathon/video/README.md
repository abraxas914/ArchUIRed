## 概述

视频模块包含两个版本：**黑客松现场版**（产品 Demo 导向）和**社交媒体版**（故事传播导向）。两个版本面向不同观众、采用不同叙事策略。

---

## 版本 A：黑客松现场视频（2 分钟）

**观众**：评委，已在摊位前，关心"能不能用、有没有价值"
**风格**：产品 screencast + 旁白，节奏紧凑
**核心目标**：用真实操作证明产品价值

### 叙事结构

#### 第一幕：共鸣（0:00–0:20）

> 画面：开发者对着屏幕，左边过时文档，右边代码，两边明显不一致

- "你有没有遇到过 — 项目知识越来越复杂，散落在代码和文档里，越积越乱"
- "AI 要生成代码，得啃一堆无关内容，效率极低"
- 关键词闪过：**知识复杂** / **难以维护** / **AI 读不动**

> 目的：20 秒内让评委点头"对，我也遇到过"

#### 第二幕：转折（0:20–0:35）

> 画面：ArchUI 节点画布出现，长文档变成模块树

- "如果把知识结构化为模块呢？每个模块独立、可复用"
- "AI 按需加载，直接生成代码"

> 目的：一句话说清解决思路，不展开细节

#### 第三幕：Demo 核心流程（0:35–1:30）— 全片高潮

三段 screencast，每段 15–20 秒：

| 段 | 展示内容 | 说明 |
|---|---|---|
| **Demo 1：模块树** | 画布上展示项目模块树，点开 SPEC → MEMORY → HARNESS | "文件系统即数据库"不是口号，是能用的 |
| **Demo 2：模块替换** | 把登录模块从"密码登录"换成"OAuth 登录"，AI agent 根据新模块重新生成代码 | **核心价值可视化** — 换个模块文档，代码跟着变 |
| **Demo 3：模块广场** | 从广场拉取支付模块，拖入画布，和现有模块连线 | 展示复用想象空间，不需要完全实现 |

> 目的：用真实操作证明"能用"，不要 PPT 式截图

#### 第四幕：为什么是我们（1:30–1:50）

> 画面：竞品对比表 or 一句话定位

- "RAG 是碎片检索，LLM Wiki 是扁平页面，Monorepo 只管代码"
- "ArchUI 从文档层就定义了建什么 — 文档即编排，模块即 spec"
- 加分项："这次黑客松的所有材料，就是用 ArchUI 自己编排的"

#### 第五幕：收尾（1:50–2:00）

> 画面：Logo + 二维码 + "Arch Park"

- "ArchUI — AI 原生生成式知识引擎 — Maintain once, generate everywhere."

### 制作要点

- **Demo 2 是全片高潮** — 花最多时间打磨这 20 秒
- **别念 PPT** — 视频用画面说话，不是读文字
- **配音节奏**：前 20s 慢（共鸣），中间快（Demo 紧凑），结尾干脆
- **字幕必须有** — 现场可能噪音大，评委可能静音看

## 版本 B：社交媒体视频（2 分钟）

**观众**：陌生人，随时划走，关心"这群人是谁、在做什么有趣的事"
**风格**：vlog / 讲述风，团队出镜，有人味儿
**核心目标**：让人记住你们、想了解更多

### 叙事结构

#### 1. 我们是谁（0:00–0:10）

> 画面：团队出镜，自然环境（工位/会议室/咖啡厅）

- "我们是 Arch Park，三个人的团队"
- 每人一句话自我介绍（姓名 + 角色，不超过 3 秒/人）

> 目的：建立人物感，让观众觉得"这几个人挺真实"

#### 2. 怎么决定做这个的（0:10–0:30）

> 画面：边说边演示痛点场景（屏幕录制穿插真人）

- "我们在用 AI 写代码的时候发现一个问题 —"
- "每次都要给 AI 喂一大堆文档，但它真正需要的可能就那几段"
- "而且文档永远是过时的，上周写的这周就对不上代码了"

> 目的：用第一人称讲真实经历，不要"行业报告"式的第三人称

#### 3. 它是什么（0:30–1:00）

> 画面：ArchUI 画布截图/录屏 + 简单动画辅助

- "所以我们做了 ArchUI — 一个 AI 原生的生成式知识引擎"
- "把知识结构化为模块，每个模块独立可维护、可复用"
- "AI 按需加载模块，直接生成代码"
- 一句话总结："让工作知识可维护、可复用、可生成，可持续产生价值"

> 目的：30 秒讲清楚是什么，用类比帮助理解

#### 4. 最酷的一个操作（1:00–1:40）

> 画面：屏幕录制，只展示 Demo 2（模块替换）

- 只做一个 Demo，但要完整、流畅、有旁白解释
- "比如我现在要把密码登录换成 OAuth 登录"
- "我只需要替换这个登录模块，AI agent 就会根据新模块重新生成代码"
- "文档改了，代码跟着更新 — 这就是我们说的'文档即编排'"

> 目的：一个操作胜过十句话，观众划到这里如果没走就赢了

#### 5. 后面还会做什么（1:40–1:55）

> 画面：路线图简图 or 团队讨论画面

- "接下来我们会做模块广场 — 把文档模块变成可以共享的资产"
- "就像 npm 分享代码包，我们分享知识模块"
- "更远的未来，不只是代码，所有创作流程都可以用模块来编排"

> 目的：让观众觉得"这个项目有未来"，想持续关注

#### 6. 收尾（1:55–2:00）

> 画面：团队合影 or Logo + 二维码

- "我们是 Arch Park，这是 ArchUI"
- "链接在评论区 / 扫码体验"

### 制作要点

- **前 3 秒决定生死** — 开头不要放 Logo 动画，直接人脸出镜说话
- **真人 > 录屏** — 录屏只用于 Demo 段，其他段尽量真人出镜
- **一个 Demo 就够** — 社交媒体不需要展示完整功能，一个最酷的操作足够
- **口语化** — "我们发现..."不要"在当今 AI 时代..."
- **节奏快** — 每个镜头不超过 8 秒，避免观众走神
- **竖屏考虑** — 如果发抖音/小红书，准备 9:16 竖屏版本

## 两版对比

| 维度 | 现场版 | 社交媒体版 |
|---|---|---|
| 时长 | 2 分钟 | 2 分钟 |
| 开场 | 痛点共鸣 | 团队自我介绍 |
| 核心 | 3 段 Demo screencast | 1 段故事 + 1 段 Demo |
| 出镜 | 无（纯录屏+旁白） | 有（团队真人） |
| 语气 | 专业、紧凑 | 真实、口语化 |
| 画幅 | 16:9 横屏 | 16:9 横屏 + 9:16 竖屏备用 |
| 投放 | 现场大屏/评委平板 | B站/YouTube/小红书/抖音 |

### 共用素材

两个版本可共用以下素材，减少重复工作：

| 素材 | 现场版用法 | 社交媒体版用法 |
|---|---|---|
| Demo 2 录屏（模块替换） | 第三幕核心 | 第四段核心 |
| ArchUI 画布录屏 | Demo 1 | 第三段辅助 |
| 竞品对比图 | 第四幕 | 不用 |
| Logo + 二维码结尾 | 第五幕 | 第六段 |
| 团队出镜 | 不用 | 第一、二、五段 |

## 技术规范

- 分辨率：1920×1080（16:9），竖屏版 1080×1920（9:16）
- 帧率：30fps
- 配音：普通话，语速适中，必须配字幕
- 背景音乐：轻音乐，不盖过人声
- 封面帧：使用 `image-design` 模块生成的主视觉
- 色彩：遵循 `vi` 模块 — 字幕颜色 `#2C1B0C`，强调色 `#C26100`（Brand Honey），辅助色 `#2F7886`（Brand Lake）
- 字体：遵循 `vi` 模块 — Sora/Noto Sans SC（字幕）+ JetBrains Mono（代码/技术术语）
