## 概述

替换流程是模块广场的核心体验闭环。用户选中项目中的模块，从广场选择一个更好的替换，系统自动处理链接重绑和 AI 代码重新生成。

## 流程步骤

```
选中本地模块 → 广场选择替换 → 预览 Diff & 确认 → AI 重新生成
```

### 1. 选中本地模块

在画布上选中要替换的模块，点击"从广场替换"按钮。

### 2. 广场选择替换

弹出广场面板，按相同标签推荐替换候选。支持样式并排对比，帮助用户做出选择。

### 3. 预览 Diff & 确认

替换前展示新旧模块的差异：

- 结构差异（子模块增减）
- 内容差异（README 内容变化）
- 链接影响（哪些链接会被重绑）

确认后执行替换：

- 后端下载新模块到本地
- 替换原有文件夹
- UUID 链接自动重绑（所有引用旧模块 UUID 的链接更新为新模块 UUID）
- 旧模块归档

### 4. AI 重新生成

替换完成后触发 LLM Sync：

- 计算 git diff
- AI agent 根据新模块 spec 重新生成对应代码
- 用户确认生成结果

## 关键技术点

- UUID 重绑：遍历项目所有 `.archui/index.yaml`，将旧 UUID 替换为新 UUID
- Diff 预览：对比两个模块树的结构和内容差异
- 归档策略：旧模块移动到 `.archui/archive/` 而非直接删除
