01 — 产品定位

两个核心价值

ArchUI Web 版围绕两个互相增强的产品支柱构建 — 画布让用户编排自己的模块，广场让用户获取和共享更好的模块。

画布编排

Canvas Editor

节点画布可视化知识模块关系。拖拽连线、无限嵌套、双击钻入。人类在画布上编排，AI agent 按需加载模块上下文生成代码。

模块广场

Module Marketplace

共享、发现、一键替换模块。同一功能多种样式 — 用户不需要从零设计，从广场选一个好看的模块替换进项目，AI 自动重新生成代码。

02 — 核心用户故事

从丑到美，一键替换

用户有一个难看的登录页面。在模块广场找到精美的登录模块，一键替换，AI 根据新 spec 重新生成代码 — 立刻得到漂亮的登录页面。

05 — 模块广场详解

共享、发现、一键替换

模块广场是 ArchUI 区别于所有竞品的关键功能。同一功能、多种样式，用户不需要从零设计。

🔍
浏览与发现
按领域/标签分类展示模块（登录、支付、鉴权、仪表盘…）。关键词搜索、热度排序、评分筛选。每个模块有结构预览 + 效果截图。
🔄
一键替换
选中本地模块 → 选择广场替换模块 → 预览 diff → 确认替换。UUID 链接自动重绑，旧模块归档。AI 根据新 spec 重新生成对应代码。
📦
发布模块
选中本地模块树 → 填写标签/分类/截图 → 一键发布到广场。支持版本迭代更新，引用方可选择是否升级到新版本。
⭐
质量筛选
使用者评分 + 评论形成社区质量信号。高下载量、高评分模块优先展示。模块作者可获得收益分成。
同一功能，多种样式 — 以登录页面为例
极简白 Minimal

06 — 技术栈

技术选型

层 选型 理由
框架 React 18 + TypeScript Spec 已指定，生态成熟
画布 React Flow 节点画布成熟方案，高度可定制
构建 Vite 极速 HMR，开发体验最佳
状态 Zustand 轻量，适合画布状态管理
样式 CSS Variables + Tailwind Token 驱动，Figma 同步友好
后端 Express 文件系统 API + 广场 API，轻量可靠
广场存储 GitHub Repo → 独立服务 初期用 Git 做 registry，后期独立
测试 Vitest + Playwright 单元测试 + E2E 全覆盖
部署 Docker / 静态 CDN 双模式：服务端部署 + 纯前端部署
07 — 黑客松 Demo 优先级

30 秒讲清核心价值

如果时间紧张，画布 + 模块替换 Demo 是最打动评委的组合。

Demo 脚本 — 核心流程
  1. 打开一个 ArchUI 项目，画布展示模块关系图 — 节点、连线、层级结构一目了然
  2. 选中一个"丑"的登录模块，右侧 Detail Panel 展示当前 spec 内容
  3. 点击"从广场替换"，打开模块广场，展示多种登录样式变体
  4. 选择一个精美的登录模块，预览 diff，确认替换
  5. 替换完成，触发 AI Sync — 根据新模块 spec 重新生成代码
  6. 展示结果：同一个项目，登录页面从丑陋变漂亮，只用了 3 次点击
