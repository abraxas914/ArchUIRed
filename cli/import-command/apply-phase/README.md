name: Apply Phase
description: Defines the execution behavior of archui import --apply, including prerequisite checks, per-module file creation, and validation gating.
---

## Overview

The apply phase executes the conversion plan generated by the preview phase. It creates identity files for each proposed module and validates incrementally to catch errors as early as possible.

## Prerequisites

Before any files are written, the apply phase runs the following checks (borrowed from `core/import/prerequisite-check`):

1. **Write access** — the target path must be writable.
2. **Git repository** — a `.git` directory must exist at or above the target path.
   - Exception: with `--partial`, this check is a warning rather than a hard stop (partial import is lower risk).
3. **Clean working tree** — no uncommitted changes.
   - If the tree is dirty, print: `Please commit or stash your changes before running archui import --apply.`

## Execution Steps

```
1. Run prerequisite checks (abort on failure unless --partial relaxes the check)
2. For each module in the proposed tree (root first, then children in order):
   a. Create the module directory if it does not exist
   b. Write README.md with name + description frontmatter
   c. Write .archui/index.yaml with generated uuid and schema_version
   d. Update parent's .archui/index.yaml submodules map
   e. Run archui validate --level 2 on the parent (fail fast)
3. If --partial: stop here (L2 target)
   If --deepen: continue to subdirectory processing (see partial-mode)
4. Run archui validate [--level 2 | .] on the full tree
5. Stage all new files with git add
6. Print summary
```

## Exit Codes

| Code | Meaning |
|------|---------|
| `0` | Apply completed, validation passed |
| `1` | Prerequisite check failed |
| `2` | Validation failed after apply (files written but tree invalid) |
